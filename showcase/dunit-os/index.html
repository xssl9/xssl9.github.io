A Rust microkernel built in the open.

Dunit OS is a small x86_64 operating system with a no_std Rust kernel, a C/NASM hardware layer, Limine boot, and a real userspace runtime — ELF execution, syscalls, and cooperative scheduling.

The system

A terminal-first OS prototype that actually runs code.

Not a mockup of a desktop. Dunit boots through Limine, drops into a framebuffer terminal, and executes real userspace processes on top of a Rust kernel.

no_std Rust kernel

PMM, VMM, kernel heap, and address-space setup, sitting on a C/NASM HAL for GDT, IDT, interrupts, and context switching.

Real ELF userspace

Userspace Rust binaries execute for real from /app, with an exec ABI that passes argc, argv, and envp, plus PATH lookup.

Process table & scheduling

Real process records with parent/child relations, wait/reap, exit codes, and a cooperative spawn / yield / resume foundation.

VFS on MemFS

A virtual filesystem rooted on an in-memory FS, with /app binaries and /assets mirroring the repository asset tree.

Recoverable faults

Userspace page faults and invalid opcodes are caught and reported as fault status instead of taking down the machine.

Honest syscall ABI

read/write/open/close, framebuffer drawing, spawn/wait, cwd/chdir, sleep, readdir/stat, IPC, stdin, and cooperative yield.

Terminal mode

The reliable development path.

A framebuffer-backed kernel terminal with command parsing, history, and autocomplete. Spawn userspace children, yield to them, resume, and wait for real exit or fault status — all from a handful of honest system commands.

  • PATH lookup through /app for exec and spawn
  • Foreground stdin can feed line input to apps
  • Minimal /proc, /dev, and ramblk0 diagnostics

Roadmap

Early, but moving with intent.

01

Userspace Runtime v1

Stabilize spawn / yield / wait / exit / fault / stdin / stdout / ipc as a runtime contract.

02

Runtime Contracts

Tighten syscall error codes, expand stdin beyond EOF, and improve ps introspection.

03

Filesystem Growth

Move beyond embedded MemFS toward a disk-backed path with mount / unmount semantics.

04

GUI Later

Revisit GUI mode with real userspace processes once scheduler and runtime contracts are strong.

Not implemented yet

Timer preemption · SMP · disk-backed FS · network stack · userspace shell process · full libc · ACPI shutdown · real RTC.
